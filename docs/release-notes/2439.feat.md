{attr}`anndata.settings.zarr_write_format` is now 3 and {func}`anndata.settings.auto_shard_v3` is now `True` (with `None` removed as an option). This means that unless the user changes one of these flags or explicitly creates a {class}`zarr.Group` with `zarr_format=2` or overrides the `shards` in `dataset_kwargs` passed in to {func}`~anndata.io.write_elem`, sharded zarr v3 stores will be written with a target uncompressed shard size of 1GB.
